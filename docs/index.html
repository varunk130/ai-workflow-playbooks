Design Principles
Built for Production
01

Workflows, Not Wikis

Each playbook is a step-by-step process agents execute, not a reference article they browse.

02

Guardrails Over Guidelines

Exit criteria must be met with evidence. Promises don't count.

03

Anti-Pattern Inoculation

Every playbook documents common shortcuts and why they fail.

04

Progressive Loading

Core playbook stays focused. Runbooks load only when needed.

05

Agent-Agnostic

Plain Markdown works with Claude Code, Cursor, Copilot, Gemini, or any agent.

06

Production-First

The pipeline extends through deployment into operations and observability.
